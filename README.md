<h1>osTicket - Ticket Lifecycle</h1>

This tutorial outlines the lifecycle of a ticket from intake to resolution within the open-source help desk ticketing system osTicket.<br />

<h2>Environments and Technologies Used</h2>

- Microsoft Azure (Virtual Machines/Compute)
- Remote Desktop
- Internet Information Services (IIS)

<h2>Operating Systems Used </h2>

- Windows 10</b> (21H2)

<h2>Ticket Lifecycle Stages</h2>

- Intake
- Assignment and Communication
- Working the Issue
- Resolution

<h2>Lifecycle Stages</h2>
<h3>1. Intake</h3>

At this level we are able to open a New Ticket
  - Populate unique user information such as Name and Email Address
  - Then, we can enter ticket summary and details and finally selct 'Create Ticket'

<h3>2. Assignment and Communication</h3>

- Tickets may be assigned/re-assigned by a specific user/agent with proper privileges. 
- Agents are able to communicate with eact other during the solution process

<h3>3. Working the Issue</h3>

- The agent that is assigned the ticket addresses the issue in a timely manner, considering SLA, and chooses to resolve it if the working of the issue actually resolves the problem or satisfies the person who opened the ticket.

<h3>4. Resolution</h3>

- Tickets are marked as resolved when working them concludes.
- At this point a ticket is marked as resolved/closed. Indicating that the problem has been correctly worked and does not exist anymore.
